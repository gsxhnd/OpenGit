# Puck 整体工作台 UI 提示词

## 目标

为 Puck 设计一套专业、高信息密度、跨平台桌面工作台 UI。Puck 面向开发者和运维人员，核心功能包括 SSH 远程、本地终端、FTP/SFTP 文件传输、远程文本编辑、远程资源监控。远期扩展 WebDAV/S3 存储、VNC/RDP 远程桌面、Docker/Kubernetes。

## 设计方向

采用"专业工作台"风格，参考 IDE、数据库客户端、云资源管理器和终端工具的组合体验。界面应强调效率、稳定、状态可见和多任务并行，而不是营销型首页或大面积卡片式 SaaS 控制台。

## 整体布局

设计一个桌面应用主界面，包含：

- 顶部自定义标题栏：应用名称、当前工作区、全局搜索/命令面板入口、窗口控制区域。
- 左侧 Activity Bar：图标式一级导航，包含 Dashboard、Connections、Sessions、Files、Settings。
- 左侧 Primary Sidebar：展示当前功能域的资源树、连接列表、分组、过滤器。
- 中央 Workbench：多 Tab 工作区，承载终端、文件管理、远程编辑、资源监控。
- 右侧 Inspector：显示连接详情、文件属性、任务日志，可折叠。
- 底部 Status Bar：显示连接状态、当前协议、传输任务、错误数量、终端 Shell 类型、编码信息。

## 关键页面

### Dashboard

展示最近连接、快速连接入口、活跃会话、正在进行的传输任务。快速连接支持选择 SSH、FTP/SFTP、本地终端。

### Connections

以资源管理器形式展示连接配置。支持分组、标签、搜索、协议筛选。连接详情表单应支持 SSH 主机、FTP/SFTP 各自的认证字段。

### Sessions

中央区域使用 Tab 管理多个活跃会话。Tab 应显示协议图标、连接名称、状态点和关闭按钮。会话内容根据类型展示终端、文件或编辑器面板。

### Files

设计本地/远程双栏文件管理器。左侧是本地文件系统，右侧是远程文件系统。顶部提供上传、下载、刷新、新建文件夹、删除、重命名、打开编辑器按钮。底部显示传输队列。

### Remote Editor

使用 Monaco 风格编辑器区域，顶部显示远程路径、协议、保存状态、编码、换行符和文件大小。编辑器应支持深色主题、语法高亮、保存按钮和关闭按钮。

### Resource Monitor

SSH 会话内的资源监控面板。展示远程服务器 CPU 使用率、内存占用、磁盘使用率。数据以实时图表或仪表盘形式呈现，刷新频率可配置。

### Settings

设置页分区包含 Appearance、Terminal、Editor、Connections、Credentials、Shortcuts、Advanced。Terminal 设置必须包含 Windows Shell 类型：`cmd`、`PowerShell`、`WSL`。

## 视觉系统

- 默认深色主题，背景使用接近黑色的分层灰阶。
- 主强调色使用冷色系蓝/青，表示连接、选中和主要操作。
- 成功、警告、错误使用清晰但克制的状态色。
- 字体使用系统无衬线，终端和路径使用等宽字体。
- 边框和分割线细而清晰，适合长时间使用。
- 避免大面积渐变、玻璃拟态和装饰性插画。

## 交互要求

- 所有长任务必须进入任务中心或传输队列，不能只依赖 Toast。
- 连接失败、认证失败、权限不足必须在对应面板内显示可操作错误。
- 文件覆盖、删除执行前必须二次确认并展示影响范围。
- Sidebar、Inspector 和底部队列可折叠。
- 支持键盘优先操作：命令面板、快速打开连接、切换 Tab、聚焦终端、打开设置。
