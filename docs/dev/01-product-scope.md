# 1. 产品范围

## 1.1 产品定位

**Puck** 是面向开发者、运维人员和技术团队的**跨平台远程工作台**。它将远程连接、终端模拟、文件传输、远程编辑、资源监控和对象存储组织在同一个桌面应用中，减少在多个工具之间切换的成本。

使用 Electron、React 和 TypeScript 构建，支持 macOS、Windows 和 Linux。

## 1.2 核心功能域

| 功能域 | 范围 |
|--------|------|
| SSH 远程 | 主机管理、连接认证、远程 Shell、端口转发、会话状态 |
| 本地终端 | 本机 Shell 会话；Windows 支持 `cmd`、`PowerShell`、`WSL` |
| FTP/SFTP | 目录浏览、上传下载、重命名删除、属性、传输队列 |
| 远程编辑 | 文本文件远程打开、编辑、保存（Monaco Editor）、语法高亮 |
| 资源监控 | SSH 会话内远程服务器 CPU、内存、磁盘使用率实时展示 |
| WebDAV/S3 | 远程存储浏览、上传下载、对象操作（远期） |
| VNC/RDP | 远程桌面连接、会话切换、输入捕获（远期） |
| Docker/K8s | 容器/镜像管理、集群浏览、Pod 日志（远期） |

## 1.3 目标用户

- 需要通过 SSH/SFTP 管理服务器与远程文件的开发者与运维人员
- 希望在同一应用中完成终端、文件传输、远程编辑的技术团队
- 后续需要接入 WebDAV/S3 存储、VNC/RDP 桌面、Docker/K8s 运维的用户

## 1.4 产品边界

- Puck 是桌面客户端，不作为服务端网关或多人协作平台
- 凭据、密钥和连接历史默认本地保存，后续可接入系统钥匙串
- 所有协议能力必须经过 preload 白名单 API 暴露，渲染进程不直接访问 Node.js 系统能力
- 文件传输以用户可见、可确认为原则，避免默认自动覆盖

## 1.5 目标平台

| 平台 | 要求 |
|------|------|
| macOS | 原生标题栏安全区、默认 shell、DMG/ZIP 分发 |
| Windows | 支持 `cmd`、`PowerShell`、`WSL` 终端类型；Squirrel 安装器 |
| Linux | 常见 shell、DEB/RPM 打包、文件系统权限差异处理 |

## 1.6 当前状态

项目已具备：

- Electron 三进程结构、React 渲染进程、Zustand 状态管理
- 设置持久化、主题系统、国际化（中/英）
- 自定义标题栏、Toast 通知、命令面板、欢迎页
- 本地终端入口、SSH 连接流程、SFTP 文件浏览基础
- 远程文本编辑基础（Monaco Editor）

当前阶段以**工作台 UI 重构**和**终端体验统一**为优先推进方向。
