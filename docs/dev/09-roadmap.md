## 能力交付顺序（与 §3.0 一致）

与 [03-features.md](03-features.md) **§3.0 远程访问与云存储** 中的优先级一致：

1. **SSH 会话 / 终端**（xterm.js）  
2. **SFTP 文件管理**  
3. **远程文档编辑**（Monaco Editor）  
4. **Docker** → **Kubernetes**（整体早于 WebDAV）  
5. **WebDAV**  
6. **S3 兼容 API**

> **阶段对应**：§3.0 六项远程与云能力落在 **Phase 2**（SSH/SFTP/Monaco）、**Phase 4–6**（Docker/K8s、WebDAV、S3）。**Phase 3 为打包与发布**，安排在 **Phase 2 远程核心闭环之后**。**Phase 4 早于 Phase 5**，满足「Docker/K8s 先于 WebDAV」。

## 路线图原则

与 [01-overview.md](01-overview.md) **§1.2 核心理念** 一致，并落实为迭代顺序：

- **远程优先**：SSH/SFTP 为主线；**Docker / Kubernetes 先于 WebDAV**；再 **WebDAV**、**S3 兼容存储**。
- **安全默认**：凭据、主机指纹、通道与 IPC 边界见 [06-design-decisions.md](06-design-decisions.md) §6.1。
- **先安全连接，再复杂编排**：会话稳定后再做批量传输、队列策略与自动化。
- **先单机体验，再多端一致**：macOS / Windows / Linux 安装与交互对齐后，再考虑配置同步等。
- **先可观测，再智能化**：进度与日志可诊断后，再强化命令面板、AI 辅助等（不单独占阶段，按需迭代）。

## Phase 2：SSH、SFTP、远程编辑（xterm / Monaco）📋

**目标**：实现 [03-features.md](03-features.md) **§3.0** 前三项及会话基础设施；技术栈见 [02-tech-stack.md](02-tech-stack.md) **§2.3**。

### 2.1 连接与会话

| 任务 | 预估 |
|------|------|
| 主机书签、端口、用户名、密钥/密码、代理 | 2 周 |
| 主机指纹、已知主机、会话多开与切换 | 与上并行或 +1 周 |

### 2.2 SSH 终端（xterm.js）

| 任务 | 预估 |
|------|------|
| 渲染进程 **@xterm/xterm**，主进程 **node-pty**（或等价）经 IPC 双向字节流 | 2 周 |
| 与 §3.3「终端」设置项联动（字体、缓冲区等） | 迭代内 |

### 2.3 SFTP

| 任务 | 预估 |
|------|------|
| 目录树、上传/下载、权限、**断点续传**（协议允许时） | 3 周 |
| 传输队列、覆盖策略、错误重试与可观测性 | 含于上或 +1 周 |

### 2.4 远程文档编辑（Monaco Editor）

| 任务 | 预估 |
|------|------|
| **monaco-editor** 打开/保存远端文本（与 SFTP 或等价通道） | 2 周 |
| 与 §3.3「编辑器」设置项联动；大文件分块/防抖保存 | 迭代内 |

**交付物**：多主机管理；SSH 侧 **xterm** 体验；SFTP 浏览与传输；远端文本 **Monaco** 编辑闭环。
