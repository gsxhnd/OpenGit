# 9. 开发路线图

## 路线图原则

- **远程优先**：以 SSH/SFTP 为产品主线，再接入 **Docker 与 Kubernetes**（里程碑优先于 **WebDAV**），其后为 **WebDAV** 与 **S3 兼容存储**。
- **先安全连接，再复杂编排**：会话、凭据、主机指纹与加密通道稳定后，再扩展批量传输与自动化任务。
- **先单机体验，再多端一致**：桌面三端（macOS / Windows / Linux）体验对齐后，再考虑配置同步等增强。
- **先可观测，再智能化**：日志、进度与错误可诊断之后，再引入命令面板、规则引擎等效率能力。

## Phase 10: SSH 与 SFTP 📋

> 目标：落实 OpenRemote 核心定位——多平台 SSH 与 SFTP 客户端。

| 任务 | 预估 |
|------|------|
| 连接与会话管理（主机、端口、密钥/密码） | 2 周 |
| SSH 终端：渲染进程 **xterm.js**（`@xterm/xterm`）+ 主进程 PTY/流（如 `node-pty`）经 IPC 桥接 | 2 周 |
| SFTP 文件树、传输队列、覆盖策略 | 3 周 |
| 远程文本编辑：**Monaco Editor** 打开/保存远端文件（与 SFTP 或等价通道衔接） | 2 周 |

**交付物**：可管理多主机；SSH 侧具备 **xterm** 终端体验；SFTP 侧具备目录/传输能力；远端文本可通过 **Monaco** 打开与保存（能力随迭代加深）。
