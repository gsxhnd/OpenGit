# 9. 开发路线图

## 路线图原则

- **先闭环，再增强**：优先打通 `打开仓库 → 查看状态 → 暂存 → 提交 → 同步远端` 的完整工作流。
- **先单仓库，再多仓库**：单仓库体验稳定后，再扩展项目管理和跨仓库操作。
- **先安全操作，再高风险操作**：rebase、冲突解决等高风险能力必须具备预览和确认。
- **先核心 Git，再生态集成**：Git 基础能力稳定后，再接入 AI、终端、平台等扩展功能。
